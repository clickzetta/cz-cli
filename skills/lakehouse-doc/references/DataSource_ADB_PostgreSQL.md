# ADB for PostgreSQL 数据源配置指南

## 概述

ADB for PostgreSQL 是阿里云提供的一种关系型数据库服务。它基于 PostgreSQL 开源数据库，支持 SQL 和 NoSQL 数据模型，提供高性能、高可用性和弹性扩展的数据存储解决方案。通过配置 ADB for PostgreSQL 数据源，您可以实现数据的高效同步和分析，满足复杂的业务需求。

## 参数配置

配置 ADB for PostgreSQL 数据源时，需要提供以下信息，以确保成功连接到数据库服务：

* **数据源名称**：为您的 ADB for PostgreSQL 数据源指定一个唯一且易于识别的名称。
* **JDBC 连接地址**：提供 ADB for PostgreSQL 实例的 JDBC 连接地址，格式通常为 `jdbc:postgresql://host:port/database`。
* **用户名**：输入用于连接 ADB for PostgreSQL 实例的用户名。
* **密码**：提供对应用户名的数据库密码。
* **Schema**：指定默认连接的 Schema，例如 `public` 或其他您需要使用的 Schema。
* **数据源描述**：（可选）为数据源添加描述信息，以帮助您或其他管理员理解该数据源的用途或特点。

## 连接配置

在连接配置方面，您可以选择以下连接方式之一：

* **直连**：确保您输入的连接信息对应的服务在公网可访问。如果数据库实例开启了 IP 访问白名单，请确保数据集成服务的出口 IP 地址已被添加到白名单中，具体 IP 地址请联系技术支持人员。
* **通过 SSH 隧道连接**：为了提高安全性，您可以选择通过 SSH 隧道连接到 ADB for PostgreSQL。启用此选项后，需要提供以下 SSH 隧道相关的配置信息：
  * **SSH 服务器地址**：填写 SSH 服务器的 IP 地址或域名。
  * **SSH 端口**：指定 SSH 服务器监听的端口号，通常为 `22`。
  * **用户名**：填写 SSH 服务器的登录用户名。
  * **密码**：填写 SSH 服务器登录用户名对应的密码。

## 注意事项

* 确保所有提供的连接信息准确无误，且 ADB for PostgreSQL 服务是可访问的。
* 请妥善保护您的数据库凭证和 SSH 凭证，避免泄露给未经授权的人员。
* 定期检查并更新您的数据源配置，以适应数据库结构变化或新的安全要求。
* 监控数据同步任务的运行状态，以便及时发现并解决可能出现的问题。

配置完成后，您即可在数据同步任务中选择此 ADB for PostgreSQL 数据源，进行数据导入或导出操作。通过 SSH 隧道连接可以增强数据传输的安全性，特别是在处理敏感数据时。
