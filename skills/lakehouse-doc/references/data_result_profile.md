# 数据结果探查

数据结果探查功能帮助用户在SQL查询结果的基础上，快速分析字段的基础统计特征、数据分布和质量情况，适用于数据清洗、质量检查、探索性分析等场景。

## 使用探查功能

操作入口：开发功能下，点击SQL开发，并在该节点上创建一个数据查询分析的脚本。点击运行结果的数据TAB后，默认在结果页面的最右侧会呈现探查功能板块。或点击右侧「探查」功能。
![](.topwrite/assets/image_1741749786774.png =163)

> 注意：数据探查仅是针对前端呈现的结果数据做的部分探查。暂不支持超出前端执行结果部分的全量数据探查

例如：当执行结果总量为2万行，但是前端只呈现了1万行数据，那么探查结果是针对前端呈现的1万条数据的可视化探查。

探查功能会基于查询结果的字段信息，给出字段填充率以及字段数值分布的信息。

* 填充率：即统计字段里有具体数值、NULL、以及空字符的具体占比。
* 空字符：统计当前字段数值里 = “ ” 的统计占比
* Null：统计前字段数值里 is null的统计占比
* 有数据：统计当前字段是正常有数据的占比情况。
* 字段数值分布：按照字段类型列出数值分布情况。其中若字段是数值类型，则会给出数值的sum以及avg的信息。
