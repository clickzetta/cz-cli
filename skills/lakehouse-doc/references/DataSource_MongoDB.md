# MongoDB 数据源配置指南

## 概述

MongoDB 是一个高性能、高可用性的 NoSQL 数据库，它提供了丰富的数据模型和灵活的查询语言。通过配置 MongoDB 数据源，您可以实现与其他系统的数据同步和集成。

## 参数配置

配置 MongoDB 数据源时，需要提供以下信息以确保成功连接到数据库：

* **数据源名称**：为您的 MongoDB 数据源指定一个唯一且易于识别的名称。
* **MongoDB 实例类型**：选择您的 MongoDB 实例类型，可以是单节点、副本集或分片集群。
* **服务地址**：提供 MongoDB 数据库节点的地址，格式为 `host:port`。
* **数据库名称**：指定需要同步的数据库名称。
* **副本集名称**（如果适用）：如果是副本集实例，需要提供副本集的名称。
* **读偏好**（如果适用）：在副本集实例下，选择读偏好，例如 `primary` 或 `secondary`。
* **用户名**：如果需要认证，提供用于连接 MongoDB 的用户名。
* **密码**：提供对应用户名的数据库密码。
* **身份认证数据库名称**：（可选）创建用户时使用的数据库名称，通常是 `admin`。
* **数据源描述**：（可选）为数据源添加描述性信息，以帮助您或其他管理员理解该数据源的用途或特点。

## 连接配置

在连接配置方面，您可以选择以下连接方式之一：

* **直连**：确保您输入的连接信息在公网可访问。如果源端开启了IP访问白名单，请确保数据集成服务的出口IP地址已被加入到白名单中，具体IP地址请联系技术支持人员。
* **通过 SSH 隧道**：为了提高安全性，您可以选择通过 SSH 隧道连接到 MongoDB。启用此选项并提供 SSH 服务的 IP 地址和端口。确保您的 SSH 客户端已正确配置，并且您有权限通过 SSH 连接到 MongoDB 服务器。

## 注意事项

* 确保所有提供的连接信息准确无误，并且 MongoDB 服务是可访问的。
* 保护您的数据库凭证信息，避免泄露给未经授权的人员。
* 定期检查并更新您的数据源配置，以适应数据库结构的变化或新的安全要求。
* 监控数据同步任务的运行状态，以便及时发现并解决可能出现的问题。

完成配置后，您就可以在数据同步任务中选择此 MongoDB 数据源，进行数据的导入或导出操作。通过 SSH 隧道连接可以增强数据传输的安全性，特别是在处理敏感数据时。
