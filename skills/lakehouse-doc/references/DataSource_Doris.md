# Doris 数据源配置指南

## 概述

Doris 是一个高性能的分布式数据仓库，专为快速分析而设计。通过配置 Doris 数据源，您可以实现数据的高效导入、导出和实时分析，满足大数据时代的业务需求。

## 参数配置

配置 Doris 数据源时，需要提供以下信息以确保成功连接到 Doris 集群：

* **数据源名称**：为您的 Doris 数据源指定一个唯一且易于识别的名称。
* **JDBC 连接地址**：提供 Oracle 数据库的 JDBC 连接地址，格式通常为 `jdbc:oracle:thin:@host:port:SID` 或 `jdbc:oracle:thin:@host:port:service_name`。例如，`jdbc:oracle:thin:@oracle-host:1521:ORCL`。
* **用户名**：输入用于连接 Doris 数据库的用户名。
* **密码**：提供对应用户名的数据库密码。
* **ExternalCatalog**：（可选）提供外部目录配置，包括类型（如 MySQL、PostgreSQL 等）和相应的连接信息，例如 `mysql://user:password@host:port/database`。
* **地址类型**：选择节点类型，提供两种选项：“FE 节点地址”和“BE 节点地址”。
  * **FE 节点地址**：提供 Doris Frontend（FE）节点的地址，用于连接和管理 Doris 集群。例如，`fe-node:8030`。
  * **BE 节点地址**：提供 Doris Backend（BE）节点的地址，用于数据存储和查询处理。例如，`be-node:9050`。
* **数据源描述**：（可选）为数据源添加描述性信息，以帮助您或其他管理员理解该数据源的用途或特点。

## 连接配置

在连接配置方面，您需要注意以下事项：

* 根据选择的地址类型，确保输入的 FE 或 BE 节点地址在公网可访问。如果 Doris 集群开启了 IP 访问白名单，请确保数据集成服务的出口 IP 地址已被加入到白名单中。具体 IP 地址请联系技术支持人员获取。

## 注意事项

* 确保所有提供的连接信息准确无误，并且 Doris 服务是可访问的。
* 如果您的 Doris 集群启用了身份验证，请保护您的凭证信息，避免泄露给未经授权的人员。
* 定期检查并更新您的数据源配置，以适应集群结构的变化或新的安全要求。
* 监控数据同步任务的运行状态，以便及时发现并解决可能出现的问题。

配置完成后，您就可以在数据同步任务中选择此 Doris 数据源，进行数据的导入或导出操作。选择合适的节点类型并确保地址的正确性，可以实现高效的数据传输，提高数据处理效率。在配置过程中，请确保遵循最佳实践和安全策略，以保护您的数据安全。
