# 计算集群规格代码变更说明

随着越来越多的客户使用云器Lakehouse的计算集群，我们发现原本以2的指数倍增长的计算集群规格，可能会造成规格间隔过大、资源浪费。因此，我们修改了计算集群的规格，现在可以以更细粒度的方式规划计算集群的规格，提高资源利用效率，降低成本。

规格代码具体调整如下：

## **原计算集群规格**

从XS到5XLarge，规格增加一级，算力资源增加一倍，同时资源消耗也增加一倍：

|         |        |                           |
| ------- | ------ | ------------------------- |
| **规格**  | **简称** | **每小时消耗**（**CRU**\***时**） |
| XSMALL  | XS     | 1                         |
| SMALL   | S      | 2                         |
| MEDIUM  | M      | 4                         |
| LARGE   | L      | 8                         |
| XLARGE  | XL     | 16                        |
| 2XLARGE | 2XL    | 32                        |
| 3XLARGE | 3XL    | 64                        |
| 4XLARGE | 4XL    | 128                       |
| 5XLARGE | 5XL    | 256                       |

## **新计算集群规格**

每种类型的计算集群，均以CRU（Compute Resource Unit）表示其规格。不同类型的计算集群，由于其适合的任务和资源配置不同，可以指定的规格也不同。

各类计算集群的规格详细说明如下：

| **类型**                         | **最小规格**（**CRU**） | **最大规格**（**CRU**） | **默认值**（**CRU**） | **步长**&#xA;（规格增减的最小单位）                                                        |
| ------------------------------ | ----------------- | ----------------- | ---------------- | ----------------------------------------------------------------------------- |
| 通用型&#xA;（GP Vcluster）          | 1                 | 256               | 1                | 1（规格示例：1，2，3，4，5，6...256）                                                      |
| 分析型&#xA;（AP Vcluster）          | 1                 | 256               | 1                | 2的n次幂（规格示例：1，2，4，8，16，32，64，128，256）                                          |
| 同步型&#xA;（Integration Vcluster） | 0.25              | 256               | 0.5              | 1（规格序列如下：有0.25 CRU和0.5 CRU两个特殊的小规格，其余规格的步长为1。规格示例：0.25，0.5，1，2，3，4，5，6...256） |

每个集群每小时消耗的计算资源量为其CRU规格\*1小时，即单位为CRU\*时。例如规格为3 CRU的分析型计算集群，运行1小时的资源消耗为 3CRU \* 1小时=3CRU\*时。其费用也按照计算集群所在区域的“CRU\*时”的单价计算。

## **兼容性说明**

### 新旧计算集群规格对应关系

存量计算集群的规格会自动转换为新规格，单位时间的计算资源消耗量和费用均不会发生变化。具体的规格对应关系如下：

|         |         |          |
| ------- | ------- | -------- |
| **原取值** | **新取值** | **适用范围** |
| XSMALL  | 1       | 所有类型VC   |
| SMALL   | 2       | 所有类型VC   |
| MEDIUM  | 4       | 所有类型VC   |
| LARGE   | 8       | 所有类型VC   |
| XLARGE  | 16      | 所有类型VC   |
| 2XLARGE | 32      | 所有类型VC   |
| 3XLARGE | 64      | 所有类型VC   |
| 4XLARGE | 128     | 所有类型VC   |
| 5XLARGE | 256     | 所有类型VC   |

### SQL新建计算集群

新规格代码上线后，新建和修改计算集群的SQL语句均将暂时兼容旧的规格代码，即Vcluster\_size 参数可同时支持以数字表达或以'XSMALL、SMALL'等代码表达的计算集群规格。请尽快迁移至使用数字表达计算集群规格的方式，以避免在切换至不兼容版本后影响计算集群的正常创建。

新建集群示例代码如下：

```SQL
-- 创建计算集群
CREATE VCLUSTER [IF NOT EXISTS] vc_name
objectProperties
[COMMENT '']
[WITH PROPERRIES ("key"="value"  "key"="value" ...)]
--参数变更   
objectProperties ::=
            VCLUSTER_SIZE = NUM --0.25-256 。兼容支持 XSMALL | SMALL | MEDIUM | LARGE | XLARGE | 2XLARGE | 3XLARGE | 4XLARGE | 5XLARGE 
```

SQL端查询计算集群规格时，所有表示计算集群规格的字段（vcluster\_size, min\_vcluster\_size，max\_vcluster\_size以及current\_vcluster\_size四个字段），均转换为以数字表达的集群大小。转换规则详见上文的“新旧计算集群规格对应关系”。

## 计费说明

计算集群规格代码变更前后，同规格计算集群的资源消耗和每小时费用**不会变更**。仅涉及计费单位的变更：计算规格由"XS、S”等字母表达变更为 1CRU，2CRU的数字表达；计算资源消耗量单位由CRU变更为CRU\*时。

变更后，估算某个集群每小时的资源消耗量和费用更加简便。

无需强记“S”规格对应每小时消耗2 CRU，再乘以资源单价来估算费用。

2 CRU的计算集群，每小时就消耗“2 CRU\*时”，仅需对应单价即可计算每小时费用。
