# 权限申请

如果您需要使用他人开发的数据资源，则需要申请该资源的权限。本文为您介绍如何申请资源权限。

## 约束限制

* 当前仅支持对单个资源申请，不支持批量申请
* 一次申请工单里，支持创建多个申请资源（即申请对象、授权对象、授权时间）。审批时，只支持全部通过或全部拒绝，不支持部分操作。

## 申请入口

* 操作入口一：在「数据」功能板块内，输入关键字或直接点击搜索，进入数据搜索结果页。在结果信息流中，选择需要申请的表，点击最右侧的「申请权限」。
* 操作入口二：在「数据」功能板块内，点击数据管理。在数据管理页面下，点击目标对象后，点击对象详情页右上角的「申请权限」。

## 申请操作步骤

按照上述申请入口，选择目标对象，点击「申请权限」后，弹出数据对象权限申请弹窗。

1）申请信息

* 对象权限：即当前对象上可申请的权限点。不同的资源对象层级提供的权限类型选项不完全相同。可获得的访问权限有全部、读权限、写权限三个选项。

  * 全部：可对该资源进行所有操作，如 Insert、Select 等。
  * 读权限：仅可对该资源做“读”操作，如 Select、Read 等。
  * 写权限：仅可对该资源进行“写”操作，例如 Alter、Drop 等。

* 授予用户：支持对角色和用户两种方式授权。

  * 当选择**个人**时，可输入租户账号关键字，从下拉列表中选择其他账号进行申请，支持选择多个。
  * 当选择**用户组**时，下拉可选实例创建后默认创建的角色或通过用户自定义创建的角色（自定义角色仅支持SQL创建，具体可参考 [创建角色](CREATEROLE.md)）。

* 授权时间：当前资源对象的授权时间均为永久有效。

* 申请理由：申请资源权限的详细原因说明。

2）可点击“添加申请”，再添加一个申请资源。申请资源中需重新填写对象权限、授予用户、授权时间、申请理由。

3）申请权限后，您可在审批中心\_申请功能下查看所有自己申请的工单，包括申请内容、申请状态等，还可以进行工单撤销或催办操作。
