# PostgreSQL 数据源配置指南

## 概述

PostgreSQL 是一个功能强大的开源对象关系型数据库系统，它提供了复杂的查询、外键、触发器和视图等高级功能。通过配置 PostgreSQL 数据源，您可以实现与其他系统的数据同步和集成，以满足各种数据分析和报告需求。

## 参数配置

配置 PostgreSQL 数据源时，需要提供以下信息以确保成功连接到数据库：

* **数据源名称**：为您的 PostgreSQL 数据源指定一个唯一且易于识别的名称，例如 `PGDataSourceExample`。
* **JDBC 连接地址**：提供 PostgreSQL 数据库的 JDBC 连接地址，格式通常为 `jdbc:postgresql://host:port/database`。例如，`jdbc:postgresql://postgres-server:5432/mydatabase`。
* **用户名**：输入用于连接 PostgreSQL 数据库的用户名，例如 `dbuser`。
* **密码**：提供对应用户名的数据库密码，例如 `dbpassword`。
* **schema**：指定默认连接的 schema，例如 `public`。
* **数据源描述**：（可选）为数据源添加描述性信息，以帮助您或其他管理员理解该数据源的用途或特点。

## 连接配置

在连接配置方面，您需要注意以下事项：

* **直连**：确保您输入的连接信息在公网可访问。如果源端开启了 IP 访问白名单，请确保数据集成服务的出口 IP 地址已被添加到白名单中，具体 IP 地址请联系技术支持人员。
* **通过 SSH 连接**：为了提高安全性，您可以选择通过 SSH 隧道连接到数据源。启用此选项后，需要提供 SSH 服务的 IP 地址和端口。请确保您的 SSH 客户端已正确配置，并且您有权限通过 SSH 连接到数据源所在的服务器。同时，您需要提供以下 SSH 隧道相关的配置信息：
    * **SSH 服务器地址**：提供 SSH 服务器的 IP 地址或域名。
    * **SSH 端口**：指定 SSH 服务器监听的端口号，通常为 `22`。
    * **用户名**：提供 SSH 服务器的登录用户名。
    * **密码**：提供 SSH 服务器登录用户名对应的密码。

## 注意事项

* 确保所有提供的连接信息准确无误，并且 PostgreSQL 服务是可访问的。
* 保护您的数据库凭证信息，避免泄露给未经授权的人员。
* 定期检查并更新您的数据源配置，以适应数据库结构的变化或新的安全要求。
* 监控数据同步任务的运行状态，以便及时发现并解决可能出现的问题。

配置完成后，您就可以在数据同步任务中选择此 PostgreSQL 数据源，进行数据的导入或导出操作。通过 SSH 隧道连接可以增强数据传输的安全性，特别是在处理敏感数据时。
