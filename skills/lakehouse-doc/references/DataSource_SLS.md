# SLS(LogHub) 数据源配置指南

## 概述

SLS（Log Service），也称为 LogHub，是阿里云提供的一种实时日志服务，用于日志数据的收集、存储、查询和分析。配置 SLS 数据源可以让您的数据同步和分析工作更加高效，特别适合处理大规模的日志数据。

## 参数配置

配置 SLS 数据源时，需要提供以下信息以确保成功连接到 SLS 服务：

* **数据源名称**：为您的 SLS 数据源指定一个唯一且易于识别的名称，例如 `my_sls_source`。
* **项目**：提供 SLS 项目名称，即您在 SLS 控制台中设置的项目标识。
* **阿里云 SLS 服务地址**：填写 SLS 服务的 Endpoint 连接地址，通常是您所在区域的 SLS 服务地址。
* **阿里云 AccessKeyId**：提供阿里云账户的 AccessKey ID，用于身份验证。
* **阿里云 AccessKeySecret**：提供与 AccessKey ID 对应的 AccessKey Secret，用于身份验证。
* **数据源描述（可选）**：为数据源添加描述性信息，以帮助您或其他管理员理解该数据源的用途或特点。

## 连接配置

在连接配置方面，需要注意以下事项：

* **直连**：确保您输入的连接信息在公网可访问。如果源端开启了IP访问白名单，请确保数据集成服务的出口IP地址已被加入到白名单中，具体IP地址请联系技术支持人员。

## 注意事项

* 确保所有提供的连接信息准确无误，并且SLS服务是可访问的。
* 保护您的凭证信息，避免泄露给未经授权的人员。
* 定期检查并更新您的数据源配置，以适应项目结构的变化或新的安全要求。
* 监控数据同步任务的运行状态，以便及时发现并解决可能出现的问题。

配置完成后，您就可以在数据同步任务中选择此 SLS 数据源，进行日志数据的导入或导出操作。通过直连方式，可以实现快速的日志数据传输，提高日志处理效率。
