# 2025年07月03日 Lakehouse Studio 产品更新发布说明

本次发布（Release 2025.07.03）引入了一系列新功能、增强特性及修复内容。请注意，这些更新将分阶段逐步推送至以下区域，预计在发布日起一至两周内完成，具体时间取决于您所在的区域。

* 腾讯云上海区
* 腾讯云北京区
* 腾讯云广州区
* 阿里云上海区
* 阿里云新加坡区
* 亚马逊云科技新加坡区
* 亚马逊云科技北京区

## 不兼容改动

* 无

## 新功能特性

### 管理中心

* 计量计费：计费账单新增 DataGPT 费用项目，支持对 AI 服务成本进行透明化管理。

  ![](.topwrite/assets/image_1752131896262.png =680)

* 账户资金：针对余额不足、资源停服场景可发送提醒消息，并可自定义消息接收方式。

  ![](.topwrite/assets/image_1752131903762.png =680)

### 数据同步

* 多表实时同步：多表合并模式支持自动适配源端动态加表。

    ![](.topwrite/assets/image_1752131911838.png =680)

* 多表实时同步：在写出到 Kafka 的消息中添加主键字段名列表信息。
* 多表实时同步：写出到 Kafka 时支持按表写入 Topic 下的不同分区。
* 离线同步：SLS数据源支持通过传参指定开始时间和结束时间。
* 数据源：将数据源的管理级别从工作空间提升至服务实例，支持将数据源共享给实例下的所有工作空间，实现可见、可用。
* 数据源：Oracle 数据源 JDBC URL 支持 Service Name 模式的写法，增强连接兼容性。

### 任务开发

* 任务开发：新增组合任务节点类型，支持基于多种任务编排构建 DAG，可统一设定调度周期、参数等。

  ![](.topwrite/assets/image_1752131921358.png =680)

* 任务开发：在任务组DAG图中增加任务依赖展示。

### 调度运维

* 任务运维：下线周期任务时支持清理全部任务实例。
* 任务运维：在周期任务列表中增加“最近一次执行”筛选功能，支持按最近一次执行状态进行筛选。

### 监控告警

* 监控告警：对因上游任务暂停导致下游任务调度超时失败而产生的告警，增加抑制机制，避免误触发。
* 监控告警：在告警推送内容中，支持通过配置方式自定义适配客户侧的域名。

### 数据质量

* 支持按校验内容对数据质量校验结果进行筛选过滤，提高问题定位效率。

### 作业历史

* 作业详情页支持点击集群进行跳转，可快速查看集群详细信息。

## 优化改进

* 产品全局：完善新手引导功能链路，例如在数据上传成功后，增加到数据查询功能路径的跳转。
* 产品全局：对于在列表等位置未完全展示内容的字段，统一添加 Tooltip 提示，方便查看完整信息。
* 产品全局：优化时间选择控件中按钮的样式、提升操作体验。
* 实例首页：优化代金券显示内容，支持展示代金券的数量和余额。
* 数据上传：优化数据预览时的数据展示方式，便于理解。
* 数据同步：在数据同步任务配置界面，优化新增数据源的自动展示功能，使其在列表中自动显示。
* 数据同步：优化多表实时同步功能，解决源端 MySQL 表过大对任务配置和执行产生影响的问题。
* 任务开发：将任务和任务组分别通过不同 Tab 展示，避免相互干扰。
* 任务开发：支持调大版本对比窗口，便于查看内容对比。
* 任务开发：在新建任务菜单中，将任务分类方式调整为“数据同步”、“数据处理”等分组。
* 监控告警：优化监控规则触发条件编辑时的操作便捷性。
* 数据质量：支持快速跳转定位并查看质量规则中展示的最新校验结果详情。
* 集群管理：计算集群作业监控统计功能数据正确性提升。

## 问题修复

* 管理中心：修复管理中心新建用户后页面列表未自动刷新的问题。
* 任务开发：修复智能解析当前仅支持新增，无法更新或覆盖已有智能解析结果的问题。
* 任务开发：修复任务代码中对象名称包含引号时，智能解析无法识别匹配的问题。
* 任务开发：修复参数页面修改参数 value 保存后不生效的问题。
* 任务开发：修复任务回滚后任务状态展示错误的问题。
* 任务开发：修复任务使用任务组提供的参数时，提交后参数不生效并报错的问题。
* 任务开发：修复任务调度暂停后脚本仍可被删除的问题。
* 任务开发：修复 Shell 节点内参数与代码中参数冲突，忽略参数功能失效的问题。
* 数据目录：修复新加坡数据资产统计页面数据长时间未刷新的问题。

## 行为变更
* 数据源：数据源的管理权限由工作空间（workspace）提升至服务实例（instance），以便在账户内更方便地共享数据源并管理其连接安全。原 workspace_admin（工作空间管理员）和 workspace_dev（工作空间开发者）角色不再具备创建数据源的权限；instance_admin（服务实例管理员）和 instance_datasource_admin（服务实例数据源管理员）角色仍保留创建和管理数据源的权限。
