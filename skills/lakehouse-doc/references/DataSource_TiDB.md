# TiDB 数据源配置指南

## 概述

TiDB 是一个开源的分布式关系型数据库，它支持水平扩展和强一致性，适合构建在线事务处理和在线分析处理（HTAP）应用。通过配置 TiDB 数据源，您可以实现数据的高效同步和实时分析，满足多种业务场景的需求。

## 参数配置

配置 TiDB 数据源时，需要提供以下信息以确保成功连接到 TiDB 集群：

* **数据源名称**：为您的 TiDB 数据源指定一个唯一且易于识别的名称。
* **JDBC 连接地址**：提供 TiDB 集群的 JDBC 连接地址，格式通常为 `jdbc:tidb://host:port/database`。例如，`jdbc:tidb://tidb-server:4000/mypd`。
* **用户名**：输入用于连接 TiDB 集群的用户名。
* **密码**：提供对应用户名的数据库密码。
* **允许使用配置信息连接所有数据库/Schema**（可选）：勾选此选项，使得配置信息可以用于连接到 TiDB 集群中的所有数据库和 Schema。如果未勾选，连接将仅限于指定的数据库/Schema。
* **数据源描述**：（可选）为数据源添加描述性信息，以帮助您或其他管理员理解该数据源的用途或特点。

## 连接配置

在连接配置方面，您可以选择以下连接方式之一：

* **直连**：确保您输入的连接地址在公网可访问。如果源端开启了IP访问白名单，请确保数据集成服务的出口IP地址已被添加到白名单中，具体IP地址请联系技术支持人员。
* **通过 SSH 隧道**：为了提高安全性，您可以选择通过 SSH 隧道连接到 TiDB。启用此选项并提供以下 SSH 隧道相关的配置信息：
  * **SSH 服务器地址**：提供 SSH 服务器的 IP 地址或域名。
  * **SSH 端口**：指定 SSH 服务器监听的端口号，通常为 `22`。
  * **用户名**：提供 SSH 服务器的登录用户名。
  * **密码**：提供 SSH 服务器登录用户名对应的密码。

## 注意事项

* 确保所有提供的连接信息准确无误，并且 TiDB 服务是可访问的。
* 保护您的数据库凭证信息和 SSH 凭证信息，避免泄露给未经授权的人员。
* 定期检查并更新您的数据源配置，以适应数据库结构的变化或新的安全要求。
* 监控数据同步任务的运行状态，以便及时发现并解决可能出现的问题。

配置完成后，您就可以在数据同步任务中选择此 TiDB 数据源，进行数据的导入或导出操作。通过 SSH 隧道连接可以增强数据传输的安全性，特别是在处理敏感数据时。勾选“允许使用配置信息连接所有数据库/Schema”选项可以提供更大的灵活性，但请谨慎使用，以避免安全风险。
