# ADB for MySQL 数据源配置指南

## 概述

ADB for MySQL 是阿里云提供的分析型数据库服务，它基于 MySQL 协议，支持高并发、高可用的在线事务处理和在线分析处理。配置 ADB for MySQL 数据源可以实现数据的高效同步和分析，满足多种业务场景的需求。

## 参数配置

配置 ADB for MySQL 数据源时，需要提供以下信息以确保成功连接到 ADB 服务：

* **数据源名称**：为 ADB for MySQL 数据源指定一个唯一且易于识别的名称。
* **JDBC 连接地址**：提供 ADB for MySQL 实例的 JDBC 连接地址，格式通常为 `jdbc:mysql://host:port/database`。
* **用户名**：输入用于连接 ADB for MySQL 实例的用户名。
* **密码**：提供对应用户名的数据库密码。
* **数据源描述**：（可选）为数据源添加描述性信息，以帮助管理员理解该数据源的用途或特点。

## 连接配置

在连接配置方面，可以选择以下连接方式之一：

* **直连**：确保输入的连接信息在公网可访问。如果源端开启了IP访问白名单，请确保数据集成服务的出口IP地址已被添加到白名单中，具体IP地址请联系技术支持人员。
* **通过 SSH 隧道**：为了提高安全性，您可以选择通过 SSH 隧道连接到 ADB for MySQL。启用此选项并提供以下与 SSH 隧道相关的配置信息：
  * **SSH 服务器地址**：提供 SSH 服务器的 IP 地址或域名。
  * **SSH 端口**：指定 SSH 服务器监听的端口号，通常为 `22`。
  * **用户名**：提供 SSH 服务器的登录用户名。
  * **密码**：提供 SSH 服务器登录用户名对应的密码。

## 注意事项

* 确保所有提供的连接信息准确无误，并且 ADB for MySQL 服务是可访问的。
* 保护您的数据库凭证信息和 SSH 凭证信息，避免泄露给未经授权的人员。
* 定期检查并更新您的数据源配置，以适应数据库结构的变化或新的安全要求。
* 监控数据同步任务的运行状态，以便及时发现并解决可能出现的问题。

配置完成后，即可在数据同步任务中选择此 ADB for MySQL 数据源，进行数据的导入或导出操作。通过 SSH 隧道连接可以增强数据传输的安全性，特别是在处理敏感数据时。
