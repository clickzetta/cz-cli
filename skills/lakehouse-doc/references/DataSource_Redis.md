# Redis 数据源配置指南

## 概述

Redis 是一个开源的、基于内存的键值数据存储系统，支持多种数据结构，如字符串、哈希、列表、集合和有序集合。它以高性能、低延迟和丰富的功能特性，广泛应用于缓存、消息队列、实时分析等场景。配置 Redis 数据源可以帮助您在数据同步任务中实现高效的数据读取和写入操作。

## 参数配置

配置 Redis 数据源时，需要提供以下信息以确保成功连接到 Redis 服务：

* **数据源名称**：为您的 Redis 数据源指定一个唯一且易于识别的名称，例如 `行为数据数据源`。

* **认证用户**：提供 Redis 服务的连接用户名。如果 Redis 服务未启用 ACL（访问控制列表）认证或使用默认用户，可以留空此项。

* **访问密码**：提供 Redis 服务的连接密码。如果 Redis 服务配置了密码认证（requirepass），请填写对应的密码。如果未设置密码，可以留空此项。

* **节点模式**：选择 Redis 的部署架构模式，当前支持：

  * **Standalone**：单机模式，适用于单个 Redis 实例的场景
  * **Cluster**：集群模式，适用于 Redis Cluster 分布式集群

* **节点信息**：提供 Redis 服务的连接地址信息，包括：

  * **IP**：Redis 服务器的 IP 地址，例如 `10.10.10.1`。
  * **Port**：Redis 服务的端口号，默认为 `6379`，示例中使用 `3306`。
  * 支持通过“添加节点”按钮配置多个节点地址，适用于集群或主从架构场景。

* **数据源描述**：（可选）为数据源添加描述性信息，以帮助您或其他管理员理解该数据源的用途或特点，例如"生产环境用户行为缓存"或"实时会话数据存储"。

* **授权给工作空间使用**：选择该数据源的使用权限范围：

  * **指定工作空间**：仅授权给特定的工作空间使用
  * **全部工作空间**：授权给所有工作空间使用

## 连接配置

在连接配置方面，您需要注意以下事项：

* **直连**：确保您输入的连接信息在网络上是可达的。如果 Redis 服务开启了 IP 访问白名单或防火墙规则，请确保数据集成服务的出口 IP 地址已被加入到白名单中，具体 IP 地址请联系技术支持人员。
* **测试连通**：配置完成后，建议点击“测试连通”按钮验证连接配置的正确性，确保系统能够成功连接到 Redis 服务。
* **网络连通性测试**：建议在正式配置前，通过 `redis-cli` 或 Telnet 工具测试网络连通性，确认端口是否开放、密码是否正确。

## 注意事项

* **确保所有提供的连接信息准确无误**：确保所有提供的连接信息准确无误，并且 Redis 服务是可访问的。特别注意 IP 地址和端口号的正确性。
* **数据类型支持**：Redis 支持多种数据结构（String、Hash、List、Set、Sorted Set 等），在数据同步时需要明确源端和目标端的数据类型映射关系。
* **性能考量**：Redis 是基于内存的数据存储，大批量数据读取可能影响 Redis 服务性能。建议在业务低峰期执行数据同步任务，或使用从节点进行数据读取。
* **保护您的凭证信息**：保护您的凭证信息，避免泄露给未经授权的人员。建议定期更换密码并遵循强密码策略。
* **版本兼容性**：确认 Redis 服务版本与数据集成平台的兼容性。推荐使用 Redis 3.0 及以上版本，以获得更好的稳定性和功能支持。
* **多节点配置**：当配置多个节点时，系统将根据节点模式自动处理连接逻辑。请确保所有节点信息准确，避免数据同步异常。
* **定期检查并更新您的数据源配置**：定期检查并更新您的数据源配置，以适应架构变化或新的安全要求。
* **监控数据同步任务的运行状态**：监控数据同步任务的运行状态，以便及时发现并解决可能出现的问题。
* **键命名规范**：建议在数据同步任务中明确 Redis 键的命名模式（如前缀、分隔符等），以便准确识别和同步目标数据。

配置完成后，您就可以在数据同步任务中选择此 Redis 数据源，进行数据的导入或导出操作。通过直连方式和合理的配置，您可以充分发挥 Redis 的高性能特性，实现快速的数据传输和处理。
