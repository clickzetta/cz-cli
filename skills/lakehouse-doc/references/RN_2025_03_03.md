## 新功能特性

### 联邦查询更新【预览发布】

* **架构扩展**：云器Lakehouse支持使用External Schema在Catalog级别映射和镜像外部数据源支持查询 Hive（之前只支持 Hive 对象存储架构，本期支持 Hive HDFS 架构）。参考[EXTERNAL SCHEMA](EXTERNALSCHMEA.md)
* **增强Delta Lake格式读取**：创建外部表Delta Lake格式自动Schema推断——创建外部表时无需手动声明字段信息，系统自动解析元数据。参考[Delta Lake 外部表](delta-lake.md)

## 导入导出更新

### COPY命令增强

支持双字符CSV分隔符（如 `||`），突破原有单字符限制，提升复杂数据兼容性。参考[COPY INTO导入](copy-into-table.md)

### Pipe功能增强

1. 支持[Kafka Table Stream](pipe-kafka-table-stream.md)直连导入，通过 Table Stream 您可以实现 Exactly Once，同时支持将连接信息存储在 Connection 中

2. [Pipe SQL命令优化](pipe-syntax.md)

   * 支持通过`SHOW CREATE PIPE pipe_name`获取Pipe的DDL语句
   * Pipe DESC 结果输出优化。DESC PIPE pipe_name 显示结果字段，Pipe 任务的输入对象名称和输出对象名称，增加 Kafka 点位消费信息
   * 支持Alter命令修改Pipe导入参数如修改计算集群

3. Pipe导入对象存储数据。支持过滤`.`开头和 `_temporary` 开头的文件或者目录文件。参数`IGNORE_TMP_FILE=FALSE|TRUE`。例如：

   ```
   s3://my_bucket/a/b/.SUCCESS
   oss://my_bucket/a/b/_temporary
   oss://my_bucket/a/b/_temporary_123/
   ```

## 计算集群

**精细化资源控制**

新增[GP型计算集群单作业资源占比配置](alter-vcluster.md)，控制单作业最大资源占比 10%。避免大查询引发的资源抢占，提升集群稳定性：

```
ALTER VCLUSTER sample_vc SET QUERY_RESOURCE_LIMIT_RATIO='0.1'; 
```

## SQL语法

1. 支持创建[SQL FUNCTION](create-sql-function.md)。通过这一功能，用户能够利用SQL DDL（数据定义语言）语句来定义自定义的SQL函数，从而增强数据处理和分析的灵活性。
2. Lakehouse 正式推出[列级安全（Column-level Security）功能](dynamic-mask.md)，支持通过动态脱敏（Dynamic Data Masking）实现对敏感数据的细粒度控制。此功能允许管理员根据用户角色或属性，动态隐藏、部分显示或替换列中的敏感信息（如身份证号、信用卡号等），从而有效保护数据隐私。用户可以通过以下 SQL 语句实现动态脱敏：

```SQL
ALTER TABLE <table_name> MODIFY COLUMN <column_name> SET MASKING POLICY <policy_name>;
```

通过定义脱敏策略并将其应用于表的列，在查询运行时，屏蔽策略将应用于该列出现的每个位置，根据屏蔽策略条件、SQL 执行上下文角色或者用户来进行脱敏。

3. Lakehouse 数据存储优化功能。通过 [OPTIMIZE 命令](small_file_optimization.md)的高级参数，用户可以进行小文件合并
4. DML增强：`UPDATE` 支持 `ORDER BY ... LIMIT` 语法
5. 【预览发布】[多方言兼容](sql-parmaters.md)：通过SQLGlot集成支持PostgreSQL/MySQL/Hive/Presto等方言的部分语法

## UDF功能

新增 `cz.sql.remote.udf.lookup.policy` 配置参数,支持动态切换 UDF 与内置函数的解析优先级

```SQL
-- 策略1：优先调用内置函数（兼容传统OLAP系统行为）
SET cz.sql.remote.udf.lookup.policy = builtin_first;

-- 策略2：优先调用UDF（适配MC/Spark作业场景）
SET cz.sql.remote.udf.lookup.policy = udf_first;

-- 默认策略：强制要求UDF带Schema前缀（保持历史兼容性）
SET cz.sql.remote.udf.lookup.policy = schema_only; 
```

## SDK

### JDBC

**内网端点优化**：新增 `use_oss_internal_endpoint=true` URL 参数配置，如果您使用的服务是阿里云支持查询时强制使用 OSS 内网 Endpoint。参考[JDBC驱动](JDBC-Driver.md)

### Java SDK

实时写入接口现已完整支持 Vector 数据类型，在Java客户端使用数组类型映射。满足 AI 场景下的向量检索需求。要求clickzetta-java版本大于2.0.0。参考[Java SDK](java-sdk-refer.md)

### Python SDK

1. 支持实时写入能力：提供 `clickzetta-ingestion-python-v2` 模块（`pip install clickzetta-ingestion-python-v2`），支持数据实时写入 Lakehouse 存储
2. 支持异步提交：提供 `clickzetta-connector-python` 模块使用`execute_async()` 方法支持异步执行 SQL 查询，特别适用于长时间运行的查询
3. 支持参数绑定：提供 \`clickzetta-connector-python\` 模块使用\`execute()\` 方法支持qmark、pyformat风格的参数绑定，适用于更灵活的查询

&#x20;[Python SDK 参考](python-sdk-refer.md)

## BUG修复

* SQL命令 `SHOW PARTITION EXTENDED` 修复结果中filesize显示错误的问题
* 生成列（Generated Column）兼容性优化。修复历史版本中 Bulkload 写入生成列时的校验报错问题
* 修复使用 COPY 命令导出数据时，指定 CSV 文件格式中的 quote 不生效问题
* 联邦查询修复External Schema在Options指定Schema不生效问题
* Volume查询正则匹配不生效问题

## 行为变更

* 数据留存周期默认值调整：`data_retention_days `由7天默认调整为1天
* 为提升开发灵活性与数据管理效率，Lakehouse 在本期版本中重磅推出 **主键表 SQL 写入支持**！现在您可以直接通过标准 SQL 语句（`INSERT`/`UPDATE`/`DELETE`）操作定义了主键的表
