# Aurora PostgreSQL 数据源配置指南

## 概述

Amazon Aurora PostgreSQL 是一种与 PostgreSQL 兼容的云原生关系型数据库，它提供高性能、可扩展性和先进的安全功能。通过配置 Aurora PostgreSQL 数据源，您可以实现数据的高效同步和分析，满足企业级的数据管理需求。

## 参数配置

配置 Aurora PostgreSQL 数据源时，需要提供以下信息以确保成功连接到 Aurora PostgreSQL 数据库：

* **数据源名称**：为您的 Aurora PostgreSQL 数据源指定一个唯一且易于识别的名称。
* **JDBC 连接地址**：提供 Aurora PostgreSQL 数据库的 JDBC 连接地址，格式通常为 `jdbc:postgresql://host:port/database`。例如：`jdbc:postgresql://aurora-postgresql-host:5432/mydatabase`。
* **用户名**：输入用于连接 Aurora PostgreSQL 数据库的用户名。
* **密码**：提供对应用户名的数据库密码。
* **Schema**：指定默认连接的 Schema，例如 `public` 或其他您需要使用的 Schema。
* **数据源描述**：（可选）为数据源添加描述性信息，以帮助您或其他管理员理解该数据源的用途或特点。

## 连接配置

在连接配置方面，您可以选择以下连接方式之一：

* **直连**：确保您输入的连接信息在公网可访问。如果源端开启了 IP 访问白名单，请确保数据集成服务的出口 IP 地址已被加入到白名单中，具体 IP 地址请联系技术支持人员。
* **通过 SSH 隧道**：为了提高安全性，您可以选择通过 SSH 隧道连接到 Aurora PostgreSQL。启用此选项并提供以下 SSH 隧道相关的配置信息：
  * **SSH 服务器地址**：提供 SSH 服务器的 IP 地址或域名。
  * **SSH 端口**：指定 SSH 服务器监听的端口号，通常为 `22`。
  * **用户名**：提供 SSH 服务器的登录用户名。
  * **密码**：提供 SSH 服务器登录用户名对应的密码。

## 注意事项

* 确保所有提供的连接信息准确无误，并且 Aurora PostgreSQL 服务是可访问的。
* 保护您的数据库凭证信息和 SSH 凭证信息，避免泄露给未经授权的人员。
* 定期检查并更新您的数据源配置，以适应数据库结构的变化或新的安全要求。
* 监控数据同步任务的运行状态，以便及时发现并解决可能出现的问题。

配置完成后，您就可以在数据同步任务中选择此 Aurora PostgreSQL 数据源，进行数据的导入或导出操作。通过 SSH 隧道连接可以增强数据传输的安全性，特别是在处理敏感数据时。直连方式则提供了更直接的数据传输路径。
