# 2026年01月30日 Lakehouse Studio 产品更新发布说明

本次发布（Release 2026.01.30, v2.0）引入了一系列新功能、增强特性及修复内容。请注意，这些更新将分阶段逐步推送至以下区域，预计在发布日起一至两周内完成，具体时间取决于您所在的区域。

* 腾讯云上海区
* 腾讯云北京区
* 腾讯云广州区
* 阿里云上海区
* 阿里云新加坡区
* 亚马逊云科技新加坡区
* 亚马逊云科技北京区

## 不兼容改动
* 无

## **重大更新**
* **Data Agent 重磅首发（试用版）**
  * AI 赋能数据开发全流程，无需复杂代码，输入自然语言即可自动生成数据开发任务。
  * 实例失败时，实时诊断根因，大幅减少手工开发与运维耗时，降低使用门槛。
![](/.topwrite/assets/image_1769763770598.png)
* **MCP Server 全新上线**
  * 搭载标准化 MCP 协议接口，轻松对接第三方 AI 应用，可直接调用平台的 Lakehouse SQL 执行、任务开发、数据集成、运维诊断等能力，打破生态壁垒，拓展使用边界。具体工具能力可参考 [MCP Server操作手册](https://yunqi.tech/documents/MCPServers)
* **企业级模型管理**
  * 模型管理是企业级「模型统一调用与治理中枢」，默认集成限流控制、运行监控、权限隔离、成本管控等核心能力，支持聚合「云服务商托管模型」与「企业自有外部模型」，实现多场景下的模型调用标准化、资源管控可视化、成本归因精细化。

## **新功能特性**
### 数据同步场景全面拓展
* **多表实时同步目标端新增数据源**：新增 PostgreSQL 和 MySQL 作为目标端数据源类型。
* **多表实时同步新增源端支持**：新增支持从 TiDB 读取数据，并写入 Lakehouse。
* **多表离线同步支持增量同步**：支持基于 WHERE 条件的增量同步。


## **优化改进**
* 数据同步：
  * 脏数据报错详情升级，精准定位失败字段与原因，问题排查更高效。
  * 全量同步分组策略优化，同步速度显著提升。
* 工作空间：列表排序规则优化，直接展示您有权限的空间，信息查看更清晰。
