# 数据源管理

DataGPT 将默认添加同区域的阿里云 Lakehouse 实例作为系统内置数据源，无需手动添加。如果想添加其他区域的 Lakehouse 数据源，请参考如下操作。

## 功能概述

本功能用于添加和配置 Lakehouse 数据源连接，支持用户将 Lakehouse 数据库接入系统进行数据分析。

## 操作入口

* 位置：产品主页，左侧导航栏：管理 > 数据源

## 配置项说明

### 基本信息

| 字段名称         | 是否必填 | 说明                         |
| ---------------- | -------- | ---------------------------- |
| 数据源名称       | 是       | 用于标识数据源的唯一名称       |
| 连接串           | 是       | Lakehouse 数据库的连接地址     |
| 用户名           | 是       | 访问数据库的用户账号，示例：admin |
| 密码             | 是       | 访问数据库的用户密码           |
| 分析型计算集群名称 | 否       | 指定用于数据分析的计算集群       |

## 操作说明

1. 输入数据源名称

2. 填写数据库连接串

   ![](.topwrite/assets/image_1736899291801.png =634)

3. 输入用户名和密码

4. 填写分析型计算集群名称

5. 点击“连接测试”按钮验证配置是否正确

6. 测试通过后，点击“保存”按钮完成添加

## 注意事项

1. 数据源名称在系统内必须唯一
2. 建议在保存前进行连接测试
3. 请确保填写的用户名具有足够的数据库访问权限
4. 密码建议使用高强度密码以确保安全性

## 错误处理

| 错误类型     | 可能原因         | 解决方案                 |
| ------------ | ---------------- | ------------------------ |
| 连接测试失败 | 连接串格式错误     | 检查连接串格式是否正确     |
|              | 用户名或密码错误   | 确认账号密码的正确性       |
|              | 网络问题         | 检查网络连接是否正常       |
| 保存失败     | 数据源名称重复     | 更换一个未使用的名称       |
|              | 必填信息未完整填写 | 检查所有必填字段是否已填写 |

^
