# 云器Lakehouse对象模型概览

云器 Lakehouse 采用分层结构管理资源，以提高数据组织和访问的效率。以下是 Lakehouse 对象模型的详细说明和示例。

![](.topwrite/assets/object-hierarchy.png)

## 账户（Account）

云器的账户是代表与云器公司建立商业关系的组织或个人（通常是企业账户），用于开通云器产品服务、付费、获取服务支持等。账户在注册时将自动生成一个全局唯一的账户名，作为您在云器产品中的唯一标识。

**特点：**
- 账户负责充值、出账单、费用支付以及订购或变更云器产品。
- 所有企业或个人在云器产品上开通的服务实例及其资源对象都归属于特定账户。

**示例：**
假设您是一家名为“XYZ科技”的企业，注册云器账户后，系统为您生成的账户名为“xyztech0128”，您的账户 URL 为“xyztech0128.accounts.clickzetta.com”。

## 用户（User）

账户注册成功后，您可以创建多个用户共同使用同一个账户内的资源。不同用户间可以通过权限控制分配可使用的数据和资源。

**示例：**
在“XYZ科技”的账户下，您可以创建多个用户，如“alice”（负责数据分析）、“bob”（负责数据开发）等。您可以根据他们的职责分配不同的权限，确保数据安全和高效协作。

## 服务实例（Lakehouse Instance）

Lakehouse 服务实例是云器 Lakehouse 产品服务的载体。在开通 Lakehouse 服务时，您需要根据指定的云服务商和地域，在账户下创建服务实例。一个账户下可以创建一个或多个服务实例（当前阶段默认仅支持创建单个服务实例）。

**特点：**
- 服务实例采用统一元数据管理数据对象、计算资源、作业任务；
- 服务实例具备区域属性，其计算、数据和其他服务资源均在此云服务商的区域内；
- 不同服务实例间默认相互隔离。

**示例：**
“XYZ科技”需要在阿里云和腾讯云分别部署 Lakehouse 服务，可以在其账户下创建两个服务实例：“ali-lakehouse”和“tencent-lakehouse”。

## 工作空间（Workspace）

工作空间是用于组织 Lakehouse 资源对象（数据对象、计算资源、用户等）并提供配套数据开发能力的逻辑对象。一个服务实例下可以创建多个工作空间。工作空间之间默认隔离，用户需加入工作空间，才可使用该工作空间内的对象。同一实例下不同工作空间内的对象，可通过跨工作空间授权的方式实现共享。

**特点：**
- 工作空间用于组织和管理数据开发、计算资源等对象。
- 工作空间之间相互隔离，确保数据安全和高效协作。

**示例：**
在“ali-lakehouse”服务实例下，您可以创建两个工作空间：“data-dev”（负责数据开发）和“data-ops”（负责数据运维）。

## 计算资源（Virtual Cluster）

计算资源由多实例虚拟计算集群和运行在集群中的计算服务构成，为用户作业提供计算环境，包括 CPU、内存以及临时存储。

## Schema

在工作空间内，Schema 是一组数据库对象的命名空间。数据库对象包括表、视图等。一个工作空间内可以包含多个命名各异的 Schema。

## 表（Table）

表是格式化的二维数据表格。

## 视图（View）

视图是一种虚拟存在的表，并不在数据库中实际存在，在使用时动态生成。

## 物化视图（Materialized View）

物化视图是一种特殊的视图，相对普通视图，物化视图在数据库中实际存在，并占用存储资源。

## Lakehouse SQL任务（JOB）

Lakehouse SQL 任务是指在 Lakehouse 中通过 Web 端数据开发功能或 CLI、JDBC 连接等方式提交的 SQL 查询任务。
