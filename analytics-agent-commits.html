Commit 2 — 参数校验收紧

创建 session 必须指定 domainId，查询结果必须指定 questionId；同步新增命名规范文档

🔒 session create — domain-id 变为必填

packages/cz-cli/src/commands/analytics-agent.ts
- .option("domain-id", { type: "number",
- describe: "Domain ID" })
+ .option("domain-id", { type: "number",
+ demandOption: true,
+ describe: "Domain ID" })
变更原因

每个 session 都必须关联一个 domain，domain 决定可查询的数据范围；不传 domainId 会导致后端报错，提前在 CLI 层校验更友好。

🔒 session result — question-id 变为必填

packages/cz-cli/src/commands/analytics-agent.ts
- "result [question-id]",
+ "result <question-id>",
"Poll a text2insight question result",
- .positional("question-id", { type: "number",
- describe: "Question ID" })
+ .positional("question-id", { type: "number",
+ demandOption: true,
+ describe: "Question ID" })
变更原因

轮询结果必须知道是哪个问题（question），缺少该参数无法发起有效请求；语法从 [可选] 改为 <必填>。

Commit 3 — Analytics Agent 接口修复

补全缺失路由、修复业务错误检测、新增 domain table remove 与 session list 命令

➕

新增路由

  domains/:domainId/tables/:tableId

POST /open/session/list
🐛

业务错误检测修复

后端始终返回 HTTP 200，通过响应体中 success: false 表达业务失败，原代码未检测此状态。

新增 extractBusinessError()

兼容两种格式：
① {data: {success:false, code, message}}
② {success:false, code, message}
检测到业务错误后调用 error() 提前返回，不再误报成功。

✨

新增 CLI 命令

domain table remove
remove <domain-id> <table-id>

从指定 domain 中移除绑定的表

session list
list --domain-id <id>

列出 text2insight sessions；domain-id 必填，支持 --source-type / --source-id 过滤

executeAnalyticsCommand — 业务错误检测注入点
